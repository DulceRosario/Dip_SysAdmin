1. Monitoreo Esencial del Sistema

Mantener visibilidad sobre los recursos del servidor es la primera línea de defensa de cualquier administrador de sistemas. Aquí se detallan los comandos clave para diagnosticar rendimiento:

# Verificar el espacio en disco en formato humano
$ df -h

# Monitorear procesos en tiempo real con uso de CPU y memoria
$ htop

# Revisar el consumo de memoria RAM
$ free -m -th

El uso correcto de herramientas como htop o iotop permite identificar cuellos de botella en entornos de producción antes de que afecten a los usuarios finales.

2. Gestión de Servicios con Systemd

La mayoría de las distribuciones modernas de Linux (Debian, Ubuntu, RHEL, Rocky Linux) utilizan Systemd para la administración del ciclo de vida de los servicios del sistema.

Iniciar

systemctl start httpd

Detener

systemctl stop httpd

Estado

systemctl status httpd

Habilitar

systemctl enable httpd

3. Automatización y Crontab

La automatización de tareas repetitivas (como respaldos de bases de datos o limpieza de logs) se gestiona eficientemente mediante el demonio cron.

# Editar las tareas cron del usuario actual
$ crontab -e

# Formato básico: Min Hora DíaMes Mes DíaSemana Comando
0 2 * * * /usr/local/bin/backup_script.sh

El ejemplo anterior ejecuta un script de respaldo de manera automática todos los días a las 02:00 AM.
